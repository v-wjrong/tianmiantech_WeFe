## Build Guide
### Maven Build
- Build Commands:
    - Clean build: `mvn clean`
    - Compile project: `mvn compile`
    - Package project: `mvn package`
    - Install to local repository: `mvn install`
    - Deploy project: `mvn deploy`
- Build Process:
    - The Maven build process mainly includes the following phases:
        1. **Clean:** Deletes files generated by previous builds.
        2. **Compile:** Compiles the project's source code.
        3. **Test:** Runs unit tests.
        4. **Package:** Packages the compiled code into a distributable format (e.g., JAR, WAR).
        5. **Verify:** Validates the packaged results.
        6. **Install:** Installs the packaged files into the local Maven repository.
        7. **Deploy:** Deploys the packaged files to a remote Maven repository.
- Output Directory:
    - The packaged files are typically located in the `target/` directory, including:
        - `target/<artifactId>-<version>.jar` or `target/<artifactId>-<version>.war`: The main build artifact.
        - `target/classes/`: Compiled class files.
        - `target/test-classes/`: Test class files.
        - `target/surefire-reports/`: Test reports.
    - For multi-module projects, each submodule generates build artifacts in its respective `target/` directory.

### Adding/Modifying Dependencies
- **Maven:** Add or modify `<dependency>` elements in the `<dependencies>` section of the `pom.xml` file. For example:
  ```xml
  <dependency>
      <groupId>com.alibaba</groupId>
      <artifactId>fastjson</artifactId>
      <version>${fastjson.version}</version>
  </dependency>
  ```

### Dependency Version Management
- **Maven (`<dependencyManagement>`):**
  - Centralize dependency version management in the root POM via `<dependencyManagement>`. For example:
    ```xml
    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>com.fasterxml.jackson.core</groupId>
                <artifactId>jackson-databind</artifactId>
                <version>${jackson-databind.version}</version>
            </dependency>
        </dependencies>
    </dependencyManagement>
    ```
  - Submodules inherit version control to avoid conflicts.
  - Use properties (`<properties>`) to manage version numbers uniformly. For example:
    ```xml
    <properties>
        <hutool.version>5.8.9</hutool.version>
        <fastjson.version>1.2.83</fastjson.version>
    </properties>
    ```
